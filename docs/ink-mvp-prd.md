# Loeme Ink MVP 产品需求文档

> 状态：Draft v0.1  
> 日期：2026-07-15  
> 产品阶段：核心体验与技术可行性验证

## 1. 产品概述

Loeme Ink 是一款浏览器端实时流体海报与动态视觉工具，产品入口规划为 `/apps/ink`。

用户无需理解流体力学、Shader 或时间轴，只需选择一个起始场景、输入文字或导入 Logo、挑选配色并在画布上拖动，即可生成彩色墨水、烟雾和光流绕过内容、填充轮廓或逐渐显现的动画，并输出静态海报或短视频。

产品核心不是提供通用动画编辑能力，而是把一种计算密集、难以制作的视觉效果压缩成一个可控、即时且可导出的创作流程。

> **Paint with motion.**  
> 用手势绘制运动，而不是逐帧制作动画。

## 2. 核心产品假设

MVP 需要验证三个假设：

1. 用户是否会因为实时流体反馈而愿意主动拖动、尝试和重复创作。
2. 用户是否能在不理解专业参数的情况下，于 3 分钟内完成一段可分享的动态视觉。
3. “预设 + 文字关系 + 手势 Take + 导出”是否足以构成独立产品，而不依赖完整时间轴、图层合成或 AI 生成。

## 3. 用户问题

现有动态图形工作流通常存在以下问题：

- 在 After Effects、Blender 或 TouchDesigner 中制作流体效果，学习和搭建成本较高。
- Shader Playground 容易产生漂亮效果，但参数技术化、作品难以控制和导出。
- 常见在线海报工具能够制作基础动画，但流体、烟雾和复杂粒子效果大多是固定模板。
- 设计师希望让文字、Logo 和品牌色“动起来”，但不想学习节点、表达式或逐帧动画。
- 实时实验结果常常只能观看，无法复现、录制或继续修改。

## 4. 目标用户

### 4.1 核心用户

- 平面、品牌和社交媒体设计师。
- 音乐人、活动主办方和内容创作者。
- 希望制作动态封面、标题卡和循环背景的独立创作者。
- 对生成艺术感兴趣，但不熟悉 Shader 或专业动画软件的用户。

### 4.2 典型输出场景

- 音乐封面和动态专辑视觉。
- 展览、演出和活动海报。
- 品牌 Logo 动效与发布片头。
- 社交媒体竖屏短动画。
- 网站 Hero 背景和静态封面。
- 演示文稿、直播和舞台背景素材。

## 5. MVP 产品目标

### 5.1 核心目标

1. 用户打开项目后 10 秒内看到一个正在运动、可以立即干预的有效场景。
2. 用户在 30 秒内理解“拖动会注入颜色并推动流体”。
3. 用户在 3 分钟内完成第一个包含文字、配色和运动的可导出作品。
4. 用户能够记录一次手势表演，重播并选择满意版本。
5. 用户能够可靠导出静态 PNG 和短视频文件。

### 5.2 产品验证指标

| 指标 | MVP 目标 |
|---|---|
| Time to First Interaction | 中位数不超过 20 秒 |
| Time to First Export | 中位数不超过 3 分钟 |
| 首次任务完成率 | 70% 以上用户无需指导完成预设到导出 |
| Take 完成率 | 60% 以上首次用户至少录制一次 Take |
| 导出成功率 | 支持环境中 PNG 98% 以上、视频 95% 以上 |
| 性能底线 | 推荐桌面设备默认场景稳定达到 30 FPS 以上 |
| 输入反馈 | 支持设备上 Pointer 到画面响应低于 50 ms |
| 首屏空白率 | 0%；任何新项目均以有效场景开始 |

## 6. 产品核心原则

1. **Motion first**：进入项目即看到运动，不从空白画布或参数面板开始。
2. **Gesture over timeline**：首版用手势 Take 表达时间，不引入专业关键帧时间轴。
3. **Designed constraints**：参数少而有效，每个控制都应产生可感知的视觉差异。
4. **Text is part of the flow**：文字不是覆盖在流体上方的装饰，而是能阻挡、容纳或显现流体。
5. **Beautiful by default**：预设必须经过人工调校，随意拖动也不容易得到脏乱结果。
6. **Local first**：模拟、录制和项目保存默认在浏览器本地完成。
7. **Honest performance**：根据设备能力自动降级，不用卡顿换取虚假的高质量标签。

## 7. 核心概念模型

Ink 不使用节点作为默认产品语言。MVP 的项目由以下概念组成：

```text
Project
├── Format：画布比例、尺寸与背景
├── Fluid：流速、旋涡、扩散和消散
├── Ink：颜色、注入方式与画笔
├── Subject：文字或 Logo 及其与流体的关系
├── Forces：手势、方向风或旋转力
├── Take：一次有时长的手势记录
└── Output：静态帧或动态视频
```

### 7.1 Take 定义

Take 是用户从初始状态开始录制的一次 3–10 秒表演，记录：

- 手势位置、速度、压力替代值和时间。
- 画笔类型与颜色切换。
- 录制期间触发的有限参数变化。
- 初始场景、预设、文字与参数快照。

Take 不保存逐帧像素或完整 GPU 状态。它用于重播和方案比较，但 MVP 不承诺不同 GPU、浏览器或系统之间逐像素一致。

## 9. 默认用户流程

```text
进入 Ink
→ 自动载入正在运动的起始场景
→ 在画布拖动，立即注入颜色与速度
→ 选择一个视觉预设
→ 输入标题文字
→ 选择文字与流体的关系
→ 调整配色和少量 Flow 参数
→ Record 一次 3–10 秒 Take
→ 重播并保存满意 Take
→ 冻结静态帧或导出短视频
```

新项目不得展示纯黑、纯白或完全静止的空画布。

## 10. 首次体验与魔法时刻

### 10.1 起始场景

默认场景包含：

- 深色背景。
- 3 种已经调和的 Ink 颜色。
- 中央文字 `MOVE WITH IT`。
- 文字模式为 Obstacle，流体会绕开字形。
- 低速持续流动，避免无人操作时完全静止。
- 画布提示：“Drag to pour ink”。

### 10.2 首个魔法时刻

用户第一次拖过文字边缘时，颜色应被手势带动并沿字形轮廓分流，在文字背后形成清晰涡旋。反馈必须在一帧内开始出现，不等待提交或重新计算。

### 10.3 渐进披露

- 第一次拖动后隐藏操作提示。
- 第一次输入文字后提示切换 Block / Fill / Reveal。
- 第一次录制后突出 Replay 和 Save Take。
- 高级 Flow 参数默认收起。

## 11. 功能需求

### 11.1 起始预设

MVP 提供 8 个经过人工调校的预设：

| 预设 | 视觉特征 |
|---|---|
| Ink Bloom | 墨滴扩散与边缘卷曲 |
| Aurora | 柔和长条光流 |
| Marble | 高对比液态纹理 |
| Smoke Type | 烟雾绕过文字 |
| Neon Current | 深色背景与发光色流 |
| Soft Cloud | 低速、低对比云状混合 |
| Vortex Poster | 中央旋涡海报构图 |
| Chromatic Spill | 多颜色高能量喷溅 |

选择预设会替换 Fluid、Ink、背景和默认画笔参数，但不得覆盖用户已经输入的文字，除非用户明确选择“Reset all”。

### 11.2 画布格式

提供以下快捷格式：

- Square：1080 × 1080。
- Portrait：1080 × 1350。
- Story：1080 × 1920。
- Landscape：1920 × 1080。
- Custom：用户输入宽高，MVP 限制在 512–4096 px。

模拟分辨率与导出分辨率分离。改变输出尺寸不得直接按同等比例增加流体计算网格。

### 11.3 画笔与手势

MVP 提供三种画笔：

| 画笔 | 行为 |
|---|---|
| Pour | 同时注入颜色与推力，默认工具 |
| Push | 只推动已有流体，不添加颜色 |
| Clear | 局部移除颜色，不清除速度场 |

高频控制：

- Size：作用半径。
- Power：注入力度。
- Ink：当前颜色。
- Flow with stroke：是否根据手势速度决定推力。

交互要求：

- 鼠标、触控板和单指触摸均可使用。
- 快速手势形成更强推力，但必须设置上限，避免模拟爆炸。
- Pointer 离开画布时结束当前笔画。
- 支持撤销最近一次完整笔画，而不是撤销单个采样点。

### 11.4 Ink 与配色

- 每个项目支持 1–5 个 Ink 颜色。
- 提供人工调校 Palette。
- 支持添加、删除、排序和修改颜色。
- 自动在多颜色间按笔画切换，或锁定当前颜色。
- 设置 Background Color。
- 提供 Blend Character：Clean / Rich / Muddy 三档，内部映射颜色混合和消散策略。

MVP 不模拟真实颜料的光谱混色，也不承诺印刷颜色准确性。

### 11.5 Flow 控制

默认只展示六个设计语言参数：

| 产品参数 | 用户感知 | 内部可能映射 |
|---|---|---|
| Speed | 流动速度 | 时间步长与速度缩放 |
| Swirl | 涡旋强度 | Vorticity confinement |
| Spread | 墨水扩散范围 | Dye diffusion |
| Fade | 颜色消散速度 | Dye dissipation |
| Drag | 流动阻力 | Velocity dissipation / viscosity |
| Turbulence | 自然扰动 | Noise force |

Advanced 中可以提供 Simulation Quality 和 Pressure Accuracy，但不直接暴露求解迭代、网格尺寸等术语。

### 11.6 Subject：文字或 Logo

MVP 支持一个主 Subject，类型可以是 Text 或 Logo。

Text 支持：

- 输入单行或多行文字。
- 使用 6–8 款随应用打包、授权明确的字体。
- Font、Weight、Size、Line Height、Tracking、Alignment。
- 在画布中移动、缩放和旋转。
- 文字颜色和基础透明度。

Logo 支持：

- 导入本地 SVG 或透明 PNG。
- 上传文件最长边限制为 4096 px。
- SVG 经安全清理后栅格化为模拟 Mask，不直接执行脚本、滤镜或外部引用。
- 细线和小型负空间可能低于模拟网格精度，导入后显示可用性检查。
- 最终合成使用原始高分辨率 Logo，模拟 Mask 的分辨率不得降低最终边缘质量。

文字与流体关系提供三种模式：

| 模式 | 行为 |
|---|---|
| Block | 文字作为障碍物，流体绕开字形 |
| Fill | 只在字形内部显示流体 |
| Reveal | 流体经过时逐渐显现文字 |

MVP 不支持逐字符动画、沿路径文字、可变字体轴动画，也不支持 Text 与 Logo 同时作为两个独立 Subject。

### 11.7 基础持续力与旋涡

除用户手势外，场景可以启用一种全局持续力：

- Directional Flow：统一方向风。
- Drift：缓慢噪声漂移。

场景还可以放置最多 3 个 Vortex：

- 点击画布放置。
- 拖动中心调整位置。
- 拖动外环调整作用范围。
- 设置顺时针 / 逆时针和力度。
- 编辑时可显示方向、范围和简化流线；预览与导出时隐藏控制标记。

MVP 不提供任意力场节点连接，也不支持超过 3 个局部力场。

### 11.8 Take 与 Transport

Transport 提供：

- Play / Pause。
- Restart：回到 Take 初始状态。
- Record：开始 3、5、8 或 10 秒录制。
- Countdown：Off / 3 秒。
- Replay。
- Save Take。
- Delete Take。

每个项目最多保存 6 个 Take。保存 Take 时生成静态缩略图并记录场景快照。

录制期间：

- 参数面板保持可见，但只允许修改已声明可录制的高频参数。
- 禁止修改画布尺寸、字体文件和模拟质量。
- 时间结束后自动停止并进入 Replay。
- Escape 取消本次录制，不覆盖已有 Take。

Loop Preview 可以重复播放 Take，但 MVP 不承诺首尾逐帧无缝。严格无缝循环属于后续能力。

### 11.9 Reset 与 Undo / Redo

- Undo / Redo 覆盖参数、文字变换、颜色和完整笔画。
- 模拟运行中的连续帧不进入历史栈。
- Reset Flow：清除颜色和速度，保留设计参数。
- Restart Take：恢复 Take 初始状态。
- Reset Scene：恢复所选预设初始状态，必须二次确认。

### 11.10 静态导出

支持：

- 当前帧 PNG。
- 透明背景 PNG；仅在当前预设和文字模式支持时开放。
- 1× / 2× 输出，最高边限制为 4096 px。
- 导出前显示尺寸、背景和预计内存提示。

静态导出应以输出尺寸重新渲染颜色结果，不要求以输出尺寸重新执行完整流体模拟。

### 11.11 动态导出

MVP 支持：

- WebM。
- 720p / 1080p。
- 30 FPS；支持环境可开放 60 FPS。
- 3–10 秒。
- 有背景 / 透明背景取决于浏览器编码能力。
- 实时录制，导出期间显示进度并保持页面前台。

产品必须先检测编码能力，再展示可用格式。不得在不支持的浏览器中展示一个必然失败的 MP4 选项。

MP4、GIF、PNG 序列、4K 视频和后台云端渲染不属于 MVP。

### 11.12 项目保存

- 自动保存在本地浏览器。
- 保存最近 10 个项目。
- 项目包含参数、文字、Palette、Take 手势数据和缩略图。
- 不保存逐帧视频或完整 GPU 缓冲区。
- 支持导出 / 导入 Loeme Ink 项目文件。
- 清理浏览器数据会删除本地项目，产品需明确提示。

## 12. MVP 边界

### 12.1 包含

- WebGPU 2D 流体模拟。
- 8 个起始预设。
- Pour / Push / Clear 三种手势工具。
- 最多 5 个 Ink 颜色和 Palette。
- 一个文字或 Logo Subject。
- Block / Fill / Reveal 三种文字关系。
- 一种持续力。
- 3–10 秒 Take 录制、重播和保存。
- PNG 与 WebM 导出。
- 本地自动保存和项目文件。
- 性能分级、设备丢失恢复和 WebGPU 不可用提示。

### 12.2 不包含

- 通用图层系统和任意合成模式。
- 关键帧、曲线编辑器和完整时间轴。
- 多段视频剪辑。
- 3D 流体、真实水面和刚体物理。
- 自定义 Shader 或代码节点。
- 通用图片背景、视频和摄像头输入；作为 Subject 的本地 SVG / PNG Logo 除外。
- 麦克风或音频响应。
- AI 图像、Palette 或动画生成。
- SVG 路径障碍物和多文字图层。
- 严格无缝循环。
- MP4、GIF、4K 和云端渲染。
- 云项目、多用户协作和模板市场。

## 13. 性能与兼容性需求

### 13.1 性能分级

首次启动执行短时能力检测，选择：

| 等级 | 建议模拟网格 | 目标 |
|---|---:|---|
| High | 512 | 推荐桌面设备 45–60 FPS |
| Medium | 256–384 | 普通集成显卡 30–60 FPS |
| Low | 128–192 | 移动设备或低功耗设备至少 30 FPS |

具体网格由技术验证确定，产品界面仅显示 High / Balanced / Efficient。

### 13.2 自动降级

出现持续掉帧时按以下顺序降级：

1. 降低模拟分辨率。
2. 降低压力求解精度。
3. 关闭高成本 Glow 或额外后处理。
4. 降低预览帧率，但保持输入采样连续。

导出尺寸不得反向影响实时预览等级。

### 13.3 WebGPU 不可用

WebGPU 不可用时：

- 不进入损坏或纯黑工作台。
- 展示短视频或静态示例说明效果。
- 说明推荐浏览器和设备要求。
- 允许查看模板，但禁用编辑和导出。
- 提供重新检测按钮。

MVP 不实现功能不一致的 Canvas 2D 流体替代版本。

### 13.4 设备丢失

GPU Device Lost 时：

- 立即暂停模拟和录制。
- 保留项目参数与 Take 数据。
- 尝试重新初始化一次。
- 成功后从 Take 初始状态恢复，不声称恢复丢失的瞬时帧。
- 失败后提供重载和项目文件导出路径。

## 14. 错误与空状态

| 场景 | 产品响应 |
|---|---|
| WebGPU 不可用 | 展示兼容说明和示例，不显示黑屏 |
| 模拟初始化失败 | 显示具体阶段、重试和低质量模式 |
| 视频编码不支持 | 隐藏不可用格式，保留 PNG 导出 |
| 导出中断 | 保留项目，不生成损坏文件，允许重试 |
| 字体加载失败 | 回退到内置默认字体并提示 |
| 项目文件损坏 | 不覆盖当前项目，说明无法恢复的字段 |
| 本地存储已满 | 停止自动保存视频缓存，允许导出项目文件 |
| 设备性能不足 | 自动切换 Efficient，并说明可手动恢复 |

## 15. 功能优先级

### P0：证明核心体验

- 有效起始场景。
- 单一人工调校流体预设。
- Pour 手势及即时反馈。
- 3 个 Ink 颜色。
- 一个文字 Subject 的 Block 模式。
- Play / Pause / Reset。
- 5 秒 Take 录制与重播。
- PNG 导出。
- WebGPU 检测、错误提示和性能计数。

### P1：形成完整 MVP

- 8 个预设。
- Push / Clear。
- 完整 Palette 与 1–5 色 Ink。
- Fill / Reveal。
- SVG / PNG Logo Subject。
- 最多 3 个可编辑 Vortex。
- 三类持续力。
- 3 / 5 / 8 / 10 秒 Take。
- 最多 6 个 Saved Takes。
- WebM 导出。
- Format 预设、项目保存、Undo / Redo。
- 自动性能分级和 Device Lost 恢复。

### P2：MVP 后验证

- 图片与 SVG Mask。
- 音频文件和麦克风响应。
- 多文字 Subject。
- 简化关键帧。
- MP4 与 4K 导出。
- 严格无缝循环工具。
- 可分享的只读作品链接。
- 模板发布和社区画廊。

## 16. 验收场景

### 场景 A：零基础首次创作

1. 用户进入新项目。
2. 画布立即显示正在运动的有效场景。
3. 用户拖动画布，墨水即时响应。
4. 用户更改三种颜色中的一种。
5. 用户导出 PNG。

验收：无需打开 Advanced，3 分钟内完成；无空白画布或技术术语阻断。

### 场景 B：文字绕流海报

1. 用户输入两行标题。
2. 选择 Block。
3. 墨水在字形边缘分流且不穿过文字主体。
4. 用户移动和缩放文字，障碍物在可接受延迟内更新。
5. 导出的 PNG 与预览构图一致。

### 场景 C：录制 Take

1. 用户选择 5 秒和 3 秒倒计时。
2. 倒计时结束后执行两次手势并切换一次 Ink 颜色。
3. 系统自动结束录制。
4. Replay 重建相同动作顺序和近似视觉结果。
5. 用户保存 Take，刷新项目后仍能重播。

### 场景 D：动态导出

1. 用户选择已保存的 5 秒 Take。
2. 选择 1080p / 30 FPS WebM。
3. 系统完成能力检查并开始录制。
4. 导出文件可独立播放，时长和画面比例正确。
5. 导出失败时不破坏项目或 Saved Take。

### 场景 E：低性能设备

1. 系统检测到推荐质量持续低于性能底线。
2. 自动切换 Balanced 或 Efficient。
3. 画布保持可交互，文字 Mask 和手势继续工作。
4. 用户能看到当前质量等级，但不会被底层网格参数干扰。

### 场景 F：WebGPU 不可用

1. 用户在不支持的环境打开 Ink。
2. 页面展示效果示例、原因和推荐环境。
3. 不显示纯黑画布，不允许开始一个无法保存的项目。
4. 用户可以重新检测或返回 Loeme 首页。

## 17. 主要风险与产品对策

| 风险 | 对策 |
|---|---|
| 效果技术上正确但不够漂亮 | 预设由设计目标反推参数；先做视觉黄金样例，再扩功能 |
| 用户只玩十秒，没有创作目标 | 文字 Subject、格式模板和 Take 把玩具转为作品流程 |
| 参数过多形成 Shader Playground | 高频参数限制为六个，底层参数只在 Advanced 出现 |
| 动态结果难以复现 | Take 保存输入事件和场景快照，承诺视觉复现而非逐像素一致 |
| 视频导出跨浏览器不一致 | 先做能力检测；PNG 永远作为可靠退路；MVP 只承诺 WebM |
| 高分辨率导出拖垮模拟 | 解耦模拟网格和输出画布，不以 4K 网格实时求解 |
| 文字碰撞成本过高 | MVP 限制一个 Subject，并将字形预栅格化为固定 Mask / SDF |
| 移动端发热和掉帧 | 自动 Efficient、限制录制时长、页面后台立即暂停 |

## 18. MVP 技术门

进入完整界面开发前，必须先通过一个独立技术 Spike：

1. 256 级二维流体网格在普通集成显卡稳定运行，并验证 512 级高质量档位。
2. Pointer 注入颜色与速度的延迟不可感知。
3. 文字 Mask 在 Block 模式下形成清晰、稳定的边界绕流。
4. 相同场景可以 Reset，并按 Take 数据重播出视觉近似结果。
5. 5 秒 1080p WebM 能够在支持环境中成功导出并播放。
6. GPU 初始化和运行错误能够显示为界面状态，而不是黑屏。

若文字 Block 或视频导出未通过，产品可以先以 PNG 创作玩具形式内部测试，但不得宣称完成 Ink MVP。

## 19. 实施顺序

### Phase 0：Visual & GPU Spike

- 单一流体场景。
- Pointer Pour。
- 文字 Block Mask。
- 关键性能和错误可见性。

### Phase 1：Playable Canvas

- 起始预设。
- Palette、Brush 和六个 Flow 控制。
- Play / Pause / Reset。
- PNG 导出。

### Phase 2：Take System

- 手势采样与压缩。
- Countdown、Record、Replay。
- Saved Takes 和本地项目。

### Phase 3：Creator MVP

- 8 个预设。
- Fill / Reveal。
- Format 与 WebM 导出。
- 性能分级、恢复和首次引导。

## 20. 产品决策结论

Loeme Ink MVP 应当是一款有明确输出目标的实时创作工具，而不是 WebGPU 技术展厅。

首版成立的最小闭环是：

```text
选择漂亮起点
→ 用文字赋予构图目标
→ 用手势制造一次运动
→ 保存 Take
→ 导出静态或短视频作品
```

在这个闭环得到用户验证前，不增加音频、AI、复杂时间轴、多图层或 3D 能力。
